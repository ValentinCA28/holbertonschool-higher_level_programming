Chapitre 00

Introduction au DOM

Comprendre ce qu'est le DOM et comment JavaScript s'y connecte.

Quand le navigateur charge une page HTML, il la parse et construit en mémoire un objet appelé le DOM — Document Object Model. C'est une représentation arborescente de tous les éléments de ta page.

Définition

Le DOM est une interface de programmation qui permet à JavaScript de lire, modifier, ajouter ou supprimer des éléments HTML en temps réel — sans recharger la page.

L'arbre du DOM

Chaque balise HTML devient un nœud dans l'arbre. Le nœud racine est toujours document.

document
<html>
<head>
<title>
<body>
<h1>
<p>
<div>

L'objet document

En JavaScript, tu accèdes au DOM via l'objet global document. C'est le point d'entrée pour tout.

// Le document représente toute la page HTML
console.log(document.title);     // "Ma Page"
console.log(document.URL);       // "https://..."
console.log(document.body);      // l'élément <body>
console.log(document.head);      // l'élément <head>
js
Important

Toujours placer tes scripts avant </body> ou utiliser DOMContentLoaded pour s'assurer que le DOM est prêt avant de l'utiliser.

document.addEventListener('DOMContentLoaded', () => {
  // Ici le DOM est 100% chargé et prêt
  console.log("DOM prêt !");
});

Chapitre 01

Sélectionner des éléments

Cibler précisément un ou plusieurs éléments dans le DOM.

Avant de modifier quoi que ce soit, tu dois sélectionner l'élément. Il existe plusieurs méthodes selon le sélecteur que tu utilises.

Les méthodes de sélection

// Par ID — retourne UN seul élément
const titre = document.getElementById('mon-titre');

// Par classe — retourne une HTMLCollection (live)
const cartes = document.getElementsByClassName('carte');

// Par balise — retourne une HTMLCollection
const liens = document.getElementsByTagName('a');

// ✅ Recommandé — Sélecteur CSS, retourne le PREMIER match
const el = document.querySelector('#nav .lien.actif');

// ✅ Recommandé — Retourne une NodeList de TOUS les matchs
const tous = document.querySelectorAll('.carte');
js
Bonne pratique

Préfère querySelector / querySelectorAll — ils acceptent n'importe quel sélecteur CSS et sont plus flexibles que les méthodes historiques.

Traverser le DOM

Une fois un élément sélectionné, tu peux naviguer dans l'arbre :

const el = document.querySelector('.parent');

el.parentElement       // élément parent
el.children           // enfants directs (HTMLCollection)
el.firstElementChild  // premier enfant
el.lastElementChild   // dernier enfant
el.nextElementSibling // frère suivant
el.previousElementSibling // frère précédent

Chapitre 02

Lire & Modifier le DOM

Changer le contenu, les attributs et les propriétés des éléments.

Contenu texte et HTML

const el = document.querySelector('#box');

// Lire / écrire le texte pur (sécurisé, ignore les balises)
el.textContent;
el.textContent = 'Nouveau texte';

// Lire / écrire du HTML (attention XSS !)
el.innerHTML;
el.innerHTML = '<strong>Gras</strong>';

// La valeur d'un input
const input = document.querySelector('input');
input.value;
input.value = 'Rempli automatiquement';
js
Sécurité

N'utilise jamais innerHTML avec des données venant d'un utilisateur (risque XSS). Préfère textContent ou createElement.

Attributs HTML

const img = document.querySelector('img');

// Lire un attribut
img.getAttribute('src');       // "photo.jpg"
img.getAttribute('alt');       // "ma photo"

// Modifier un attribut
img.setAttribute('src', 'new.jpg');
img.setAttribute('alt', 'nouvelle image');

// Vérifier l'existence
img.hasAttribute('disabled');  // true/false

// Supprimer
img.removeAttribute('disabled');

// Accès direct via propriété (souvent plus simple)
img.src = 'new.jpg';
img.alt = 'nouvelle image';

Chapitre 03

Styles & Classes CSS

Changer l'apparence des éléments via JavaScript.

Modifier le style inline

const box = document.querySelector('.box');

// Lire une propriété CSS (camelCase !)
box.style.backgroundColor;   // "" si pas inline

// Modifier
box.style.backgroundColor = '#f97316';
box.style.width            = '200px';
box.style.fontSize         = '1.5rem';
box.style.display          = 'none';   // cacher

// Lire le style calculé final (tous les CSS confondus)
const computed = getComputedStyle(box);
computed.getPropertyValue('color');   // "rgb(255, 255, 255)"
js

Manipuler les classes recommandé

Plutôt que de modifier le style ligne par ligne, il vaut mieux ajouter/retirer des classes CSS — c'est plus propre et maintenable.

const el = document.querySelector('.card');

el.classList.add('active');        // ajouter
el.classList.remove('active');     // supprimer
el.classList.toggle('active');     // ajouter si absent, retirer si présent
el.classList.contains('active');  // true/false
el.classList.replace('old', 'new'); // remplacer
js

Chapitre 04

Événements

Réagir aux actions de l'utilisateur : clics, frappes, survols...

Le DOM est event-driven : tout se passe en réaction à des événements. Tu attaches des fonctions (listeners) qui s'exécutent quand un événement survient.

addEventListener

const btn = document.querySelector('#mon-btn');

// Syntaxe : element.addEventListener(type, callback)
btn.addEventListener('click', (event) => {
  console.log('Cliqué !', event);
});

// Retirer un listener (la fonction doit être nommée)
function handleClick() { console.log('click'); }
btn.addEventListener('click', handleClick);
btn.removeEventListener('click', handleClick);
js

L'objet event

btn.addEventListener('click', (e) => {
  e.target;           // l'élément cliqué
  e.currentTarget;    // l'élément avec le listener
  e.type;             // "click"
  e.preventDefault(); // bloque le comportement par défaut (ex: lien)
  e.stopPropagation(); // empêche la propagation vers le parent

  // Coords de la souris
  e.clientX; e.clientY;

  // Touche clavier
  e.key;    // "Enter", "a", "Escape"...
  e.code;   // "KeyA", "ArrowUp"...
});
js

Event Delegation pattern important

Au lieu d'ajouter un listener sur chaque élément, tu en mets un seul sur le parent et tu filtres via e.target.

const liste = document.querySelector('ul');

// Un seul listener pour TOUS les <li>
liste.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    e.target.style.color = 'orange';
  }
});

Chapitre 05

Créer & Supprimer des éléments

Construire dynamiquement des éléments HTML depuis JavaScript.

Créer un élément

// 1. Créer l'élément
const li = document.createElement('li');

// 2. Le configurer
li.textContent = 'Nouvel item';
li.classList.add('item');
li.setAttribute('data-id', '42');

// 3. L'insérer dans le DOM
const liste = document.querySelector('ul');
liste.appendChild(li);         // à la fin
liste.prepend(li);             // au début
liste.insertBefore(li, refEl); // avant un élément

// Méthode moderne
liste.insertAdjacentElement('beforeend', li);
// 'beforebegin' | 'afterbegin' | 'beforeend' | 'afterend'
js

Supprimer un élément

const el = document.querySelector('.a-supprimer');

// Méthode moderne (directe)
el.remove();

// Ancienne méthode (via le parent)
el.parentElement.removeChild(el);

// Vider tous les enfants
const conteneur = document.querySelector('#liste');
conteneur.innerHTML = '';
js

Cloner un élément

const carte = document.querySelector('.card');

// true = deep clone (avec tous les enfants)
const copie = carte.cloneNode(true);
document.body.appendChild(copie);
js

Chapitre 06

Requêtes réseau (fetch)

Communiquer avec une API sans recharger la page.

L'API fetch permet de faire des requêtes HTTP asynchrones depuis le navigateur. C'est la base des applications web modernes (pas de rechargement de page).

Syntaxe de base — GET

// fetch() retourne une Promise
fetch('https://api.exemple.com/users')
  .then(response => {
    if (!response.ok) throw new Error('Erreur HTTP');
    return response.json(); // parse le JSON
  })
  .then(data => {
    console.log(data); // tableau ou objet JS
  })
  .catch(err => {
    console.error('Erreur :', err);
  });
js

Syntaxe moderne — async/await recommandé

async function getUsers() {
  try {
    const response = await fetch('https://api.exemple.com/users');

    if (!response.ok) throw new Error(`HTTP ${response.status}`);

    const users = await response.json();

    // Afficher dans le DOM
    const liste = document.querySelector('#liste');
    users.forEach(user => {
      const li = document.createElement('li');
      li.textContent = user.name;
      liste.appendChild(li);
    });
  } catch (err) {
    console.error('Erreur :', err);
  }
}

getUsers();
js

POST — Envoyer des données

async function createUser(userData) {
  const response = await fetch('https://api.exemple.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(userData),
  });
  return response.json();
}

Chapitre 07

Débogage JavaScript

Trouver et corriger les erreurs efficacement.

La console — ton meilleur ami

// Affichage
console.log('valeur', variable);     // log basique
console.warn('Attention !');         // avertissement jaune
console.error('Erreur !');           // erreur rouge
console.table(tableau);              // affiche un tableau
console.dir(element);               // inspecte un élément DOM

// Grouper des logs
console.group('Mon groupe');
console.log('A');
console.log('B');
console.groupEnd();

// Mesurer le temps
console.time('fetch');
await fetch('...');
console.timeEnd('fetch');  // "fetch: 123ms"
js

Les erreurs courantes

// ❌ Erreur : null ou undefined n'a pas de propriétés
const el = document.querySelector('#nexiste-pas');
el.textContent = '...'; // TypeError: Cannot set properties of null

// ✅ Toujours vérifier
if (el) el.textContent = '...';
// ou
el?.textContent; // Optional chaining (?.) — ES2020

// ❌ Script exécuté avant le DOM
const btn = document.querySelector('#btn'); // null si en <head>

// ✅ Attendre le DOM
document.addEventListener('DOMContentLoaded', () => {
  const btn = document.querySelector('#btn'); // OK
});
js

Les breakpoints dans DevTools

Workflow de débogage

1. Ouvre les DevTools : F12 ou Ctrl+Shift+I
2. Onglet Sources → ouvre ton fichier JS
3. Clique sur le numéro de ligne pour poser un breakpoint
4. Recharge la page — l'exécution s'arrête à ce point
5. Inspecte les variables dans le panneau Scope

// Alternative : mot-clé debugger
function maFonction(data) {
  debugger; // pause l'exécution ici si DevTools ouvert
  // ...
}
js

Erreurs fetch courantes

async function getData() {
  try {
    const res = await fetch(url);

    // fetch ne throw PAS sur une erreur HTTP !
    // res.ok est false pour 404, 500, etc.
    if (!res.ok) {
      throw new Error(`Erreur ${res.status}: ${res.statusText}`);
    }

    const data = await res.json();
    return data;
  } catch (err) {
    // Ici : erreur réseau OU notre throw
    console.error('getData échoué :', err.message);
  }
}
